## 2. Behavioral / architectural insights (deployed-code facts the audit established)
1. **`session_states` is INERT for rendering.** The render study-day and word-pool derive ENTIRELY from durable
   `class_progress.currentStudyDay` reconciled against the `attempts` anchor. `determineStartingPhase` treats **attempts
   as the sole authority** (studyService.js:228; DailySessionFlow.jsx:822-834); `session_states.phase`/`currentStudyDay`
   are non-authoritative DISPLAY CACHE (DailySessionFlow.jsx:315-320). **Implication (key safety property): a poisoned or
   impossible session CANNOT corrupt the render or falsely advance progress** вҖ” durable state stays intact. Validated live
   (choi impossible session left csd=15 untouched).
2. **`interventionLevel` is DERIVED, not stored.** The throttle hold is recomputed server-side each session from recent
   review scores via `deriveThrottleModeServer` (foundation.js:702, enter<0.30 / exit>0.50 hysteresis) /
   `calculateInterventionLevel` (foundation.js:348). A stored `interventionLevel` in a seed is overwritten on next action;
   the held precondition is encoded in `recentSessions` (low review scores), not a stored field.
3. **Reconciliation is NON-DEMOTING** вҖ” `safeCSD = max(storedCSD, resolvedCSD)` (foundation.js:1789/1833;
   progressService.js:270-272). Consequences: an inflated runaway csd does NOT self-heal (CS repair remains necessary BY
   DESIGN вҖ” demonstrated by мҳӨмңӨк¶Ң); an impossible csd exceeding durable is FLAGGED (`quarantine_candidate`) and, in
   write-capable mode, BLOCKS fail-closed вҖ” never force-reset.
4. **Off-by-one advances on COMPLETION, not bare load** вҖ” the PR-1 reconcile is completion-triggered and
   score-independent (bare login does NOT advance; a completed day does). Confirmed with a graded pass (r55).
5. **Test mode varies PER CLASS** (MCQ vs typed) вҖ” a real product variable; the assignment's `testMode` drives
   `/mcqtest` vs `/typedtest`. The harness must detect it per class.
6. **Typed-test grader is legitimate** вҖ” the test shows the English word and asks "Type your definition"; the Korean
   definition (`word.definitions.ko`) is a correct answer (TypedTest.jsx:788-790, 1722, 1739). Answering in Korean is a
   faithful student simulation, not a grader exploit; repeating the English word verbatim is correctly rejected.

## 3. Harness / methodology insights (how to drive & judge tier-3 reliably)
1. **New-word test REACH path** (no sim mode on prod вҖ” `VITE_SIMULATION_MODE` gate is false): dismiss the first-run
   "Start Studying" customize modal (always present on a fresh Playwright context) вҶ’ kebab `[aria-label="Session menu"]`
   вҶ’ **"Skip to Test"** (gated `currentQueueLength>0`) вҶ’ confirm **"Start Test"** вҶ’ navigates to the SEPARATE route
   `/mcqtest|/typedtest/<class>/<list>`. Alt: press key **`C`** per card to DRAIN (auto-opens the test at вүӨ1 card).
   The **"Next card" arrow only cycles the index вҖ” it does NOT dismiss** (the ~30s/card trap). **No `data-testid`s in
   this flow** вҖ” target by aria-label/visible text only. No flip needed before `C`.
2. **DIRECT-NAV standing rule** (choi two-list lesson): go straight to `/session/<classId>/<listId>` from the roster and
   assert `routedUrl` contains the intended `listId`; the dashboard list-picker may load a DIFFERENT assigned list.
3. **Server-path proof (M7) must be FRESH-scoped** вҖ” run `assert-recovery.mjs --since=<run-start ISO>`. Stale prior-round
   `system_logs` must not count as proof. (Fixed two bugs: a composite `userId+timestamp` query hitting an undeployed
   index and being swallowed to `[]`; and a `sinceMs=0` default that counted everything.) system_logs is keyed on
   **`userId`** (not `uid`).
4. **Step-logging is mandatory** вҖ” synchronous per-step JSONL (`lsr_step_logger.mjs`), in-loop `progress()` +
   `heartbeat()` on any >10 loop, fail-fast вүӨ25s races. Cross-machine live-tailable; a wedge leaves its last step visible.
5. **Tier model**: tier1 emulator / tier2 live-callable = server pre-filter; **tier3 live-UI Playwright = THE audit**.
6. **Verdict engine (`assert-recovery.mjs`) judges** now cover: throttle-deadlock, skip-hold, off-by-one, list-end,
   lost-save, runaway-inflated, normal-progress, canonical-anomaly, readonly-safe. Universal invariant: canonical
   list_progress must stay 0 pre-P5 (except the canonical-anomaly scenario, which seeds one by design). Every fixed-family
   PASS requires FRESH server-path proof.
