## The discipline rule (v2 — now with a hard precondition gate)
An audit only "eliminates" a risk if it **(1) faithfully reproduces the exact state that makes the risk possible**
(durable + session + browser/localStorage/sessionStorage state), **(2) asserts a machine-checkable PRECONDITION proving
that state is actually loaded** (the risky affordance is present on the *target* list), and only then **(3) drives the
risky action to completion**. A green that skipped step (2) is non-reproduction, not safety. Every recipe below carries an
explicit precondition assertion; if it fails, the run is `INVALID_PRECONDITION`, never PASS.

## 3. The three corrections that matter most (why v1 would have misled)
1. **R1 wrong vehicle.** 오윤권 is throttle-held (last-3 review avg 0.11 < 0.30 → review-only, no new test). Driving him
   green-lights "guarded" without reaching a mint. The real exposed population is inflated-**and-not-throttled** (the
   ~144 relief-reconciled). Recipe must escape-first OR seed the relief shape (nulled review scores).
2. **R2 units bug, not scope.** `csd > twi+7` compares days to words → structurally blind. Per-list alone re-scans clean.
   Needs a pace/anchor predicate. Until then the inflated pool (≥44) is uncounted, so R1's urgency is unknown.
3. **R13 is live.** MCQ correctness client-computed + anchor validation shadow-only + client scores clamped-not-validated
   = a tampered client can forge a pass today. Documented "Phase E" residual, but a real gap the register must carry.

## 4. Recommended sequence (v2)
1. **R2 predicate fix + real 26SM re-scan** → size the actual inflated pool (this decides R1 urgency).
2. **R8 functions-side success log** (emit `session_completed`), *then* the engine tighten — advance drives are ungradeable until this exists.
3. **Build harness modes:** partial-score MCQ (R6), pre-epoch-disengaged seed (R9), mid-flow storage poison (R4a),
   fabricated-`sessionContext` callable (R13-demo), deployed-rules deny-probe (R5), machine-checked precondition asserts.
4. **Drive batch** (each gated on its precondition): R1(escape-first + relief variant), R3(hold), R4a, R6, R9, R10, R11,
   R13-demo, R14, R15.
5. Route confirmed defects to code fixes (R1 screen, R13 flags, R11 grader, R14 re-entry, R3 docid).
6. Tier-2/emulator: R4b, R12, R16.
7. R7 stays a roadmap/code item (next-list carry, not D4/P5).
8. Re-run the honest audit report + the headless Fable+Opus max-effort check on the strengthened result.
