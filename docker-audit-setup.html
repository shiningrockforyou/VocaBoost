1 Prerequisites

  • Docker Desktop installed and running (Engine running in bottom-left)
  • Claude Max subscription (auth uses Max plan credentials)
  • Dev server running on your Windows host at http://localhost:5173
The audit agents connect to localhost:5173 on the host. Docker containers need --network host (Linux) or host.docker.internal:5173 (Windows/Mac) to reach it. The Dockerfile below handles this.

2 Create Writable Folders

The container mounts the project as read-only, except for the findings output directory. Create it if it doesn't exist:

PowerShell 
cd C:\Users\dmchw\vocaboost
mkdir -Force src\apBoost\criteria_audit\playwright_reports

3 Create the Dockerfile

Create Dockerfile.audit in the project root:

Dockerfile 
FROM node:20-bookworm

# System dependencies for Playwright's Chromium
RUN apt-get update && apt-get install -y \
    libnss3 libatk1.0-0 libatk-bridge2.0-0 libcups2 libdrm2 \
    libxkbcommon0 libxcomposite1 libxdamage1 libxrandr2 libgbm1 \
    libpango-1.0-0 libcairo2 libasound2 libxshmfence1 \
    fonts-liberation fonts-noto-cjk \
    git curl \
    && rm -rf /var/lib/apt/lists/*

# Install Claude Code
RUN npm install -g @anthropic-ai/claude-code

# Install Playwright with Chromium
RUN npx -y playwright@latest install --with-deps chromium

# Pre-configure Playwright MCP for the workspace
RUN mkdir -p /root/.claude && echo '{ \
  "projects": { \
    "/workspace": { \
      "mcpServers": { \
        "playwright": { \
          "type": "stdio", \
          "command": "npx", \
          "args": ["-y", "@playwright/mcp@latest", "--browser", "chromium", "--headless", "--no-sandbox"], \
          "env": {} \
        } \
      } \
    } \
  } \
}' > /root/.claude.json

WORKDIR /workspace
CMD ["bash"]
Using npm install -g instead of the native installer — more reliable in Docker.

4 Build the Image

PowerShell 
cd C:\Users\dmchw\vocaboost
docker build -f Dockerfile.audit -t claude-audit .

Takes 5-10 minutes first time (~2GB download). Subsequent builds use cache.

5 One-Time Login ONE TIME

Run a temporary interactive container to authenticate:

PowerShell 
docker run -it --rm `
  -v "$env:USERPROFILE\.claude:/root/.claude" `
  claude-audit

Inside the container:

Bash (inside container) 
claude

Copy the URL, open in your browser, log in. Once authenticated:

Bash (inside container) 
/exit
exit

Credentials are now saved to C:\Users\dmchw\.claude\ on the host.

Verify: ls $env:USERPROFILE\.claude\.credentials.json — should exist and not be empty.

6 Run a Container

PowerShell 
docker run -it --rm `
  -v "$env:USERPROFILE\.claude:/root/.claude:ro" `
  -v "${PWD}:/workspace:ro" `
  -v "${PWD}/src/apBoost/criteria_audit/playwright_reports:/workspace/src/apBoost/criteria_audit/playwright_reports:rw" `
  --add-host=host.docker.internal:host-gateway `
  --memory=6g `
  --name audit-session-1 `
  claude-audit
FlagPurpose
.claude:roCredentials mounted read-only (prevents token race conditions between containers)
/workspace:roEntire project is read-only — agent can't modify source code
playwright_reports:rwFindings directory is writable (only place agent can write)
--add-hostMaps host.docker.internal so container can reach localhost:5173 on your host
--memory=6g~6GB per container (Chrome + Node + Claude Code)
Important: The app URL inside the container is http://host.docker.internal:5173, not localhost:5173. The audit prompt template (Step 12) handles this.

7 Launch Claude Code

Inside the container:

Bash (inside container) 
claude --dangerously-skip-permissions

No login prompt. Starts immediately.

8 Verify Playwright

Inside Claude Code, type:

Claude Code prompt 
Use Playwright MCP to navigate to http://host.docker.internal:5173 and take a screenshot

Check MCP is registered:

Claude Code 
/mcp

Should list playwright with browser_navigate, browser_click, etc.

If Playwright doesn't show: Exit Claude, run claude mcp add playwright npx @playwright/mcp@latest -- --browser chromium --headless --no-sandbox, restart.

11 Fire-and-Forget Batch Script OPTIONAL

Run audit agents completely unattended. Create run-audits.ps1:

PowerShell 
# run-audits.ps1 — Fire-and-forget parallel audit runner
# Usage: .\run-audits.ps1
# Runs 3 B13 personas in parallel, waits, then runs next 3

$image = "claude-audit"
$reportsPath = "${PWD}/src/apBoost/criteria_audit/playwright_reports"

function Run-Audit($name, $prompt) {
    Write-Host "Starting $name..." -ForegroundColor Cyan
    docker run -d --rm `
      -v "$env:USERPROFILE\.claude:/root/.claude:ro" `
      -v "${PWD}:/workspace:ro" `
      -v "${reportsPath}:/workspace/src/apBoost/criteria_audit/playwright_reports:rw" `
      --add-host=host.docker.internal:host-gateway `
      --memory=6g `
      --name $name `
      $image `
      bash -c "claude -p '$prompt' --model sonnet --dangerously-skip-permissions > /workspace/src/apBoost/criteria_audit/playwright_reports/log_${name}.txt 2>&1"
}

# --- Batch 1: P1-P3 ---
Run-Audit "b13-p1" "Run audit batch B13-P1 (Speed Clicker). Read BATCH_ORCHESTRATION.md for full details. Account: student4@apboost.test / Student123!. App URL: http://host.docker.internal:5173. CRITICAL: Write findings INCREMENTALLY to findings_B13_P1.md after each scenario step."

12 Audit Prompt Template

Use this template when launching audit agents interactively. The critical rules are in the prompt so they survive context compression.

For B13 Personas (Chaos Testing)

Prompt Template 
Run audit batch B13-P[N] ([Persona Name]).

## Setup
- Read src/apBoost/criteria_audit/playwright_reports/BATCH_ORCHESTRATION.md — find B13-P[N] section
- Read src/apBoost/criteria_audit/playwright_reports/FINDINGS_TEMPLATE.md for output format
- Read src/apBoost/criteria_audit/playwright_reports/audit_state.json for test IDs

## Credentials
- Account: student[X]@apboost.test / Student123!
- App URL: http://host.docker.internal:5173

## Critical Rules
1. WRITE FINDINGS INCREMENTALLY: Create findings_B13_P[N].md with header BEFORE starting. Append each result IMMEDIATELY after each test step. Do NOT wait until the end.
2. Login first. If login fails, write it as a Blocker and stop.
3. Follow the exact scenario steps from BATCH_ORCHESTRATION.md.
4. Take screenshots at every major verification point.
5. Check console messages after each navigation.
6. When writing "How to Fix", READ the source file first — reference actual function names and line numbers.

14 Troubleshooting

Container can't reach localhost:5173

Use http://host.docker.internal:5173 instead. The --add-host flag maps this to your Windows host. Verify inside container:

Bash 
curl -s http://host.docker.internal:5173 | head -5

Login prompt appears despite mounting .claude

Credentials may have expired. Re-run the one-time login (Step 5) — mount .claude as :rw for that one container.

"Read-only file system" when writing findings

The playwright_reports volume mount may not be matching. Check the path is exact:

Bash (inside container) 
touch /workspace/src/apBoost/criteria_audit/playwright_reports/test.txt && rm /workspace/src/apBoost/criteria_audit/playwright_reports/test.txt && echo "OK"

Playwright MCP not found

Bash (inside container) 
claude mcp add playwright npx @playwright/mcp@latest -- --browser chromium --headless --no-sandbox

Container OOM killed

Increase memory limit. For 3 concurrent containers on 32GB host:

PowerShell 
--memory=8g

"Name already in use" error

PowerShell 
docker rm -f audit-b13-p1

Check running containers

PowerShell 
docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"

View logs from fire-and-forget run

PowerShell 
Get-Content src\apBoost\criteria_audit\playwright_reports\log_b13-p1.txt -Tail 50

Kill all audit containers

PowerShell 
docker ps -q --filter "name=audit-" | ForEach-Object { docker rm -f $_ }

Auth conflicts between concurrent containers

Shouldn't happen with :ro mount. If you see auth issues, all containers are reading the same token without trying to refresh it. If tokens do expire mid-run, you'll need to re-login (Step 5) and restart containers.
